## 🚧 既知の課題・技術的負債

### 🔴 優先度高
- `CharactersService` が `new PrismaClient()` を直接インスタンス化している（他 Service は DI）
- `supabase-auth.guard.ts` / `play.vue` / `MessageWindow.vue` に `console.log` が本番コードに残存（JWT ペイロード出力含む）

### 🟡 優先度中
- `characters.service.ts` に `remove`（ソフトデリート）と `deleteMyCharacter`（ハードデリート）が両方存在し意図が不明確。アセットのソフトデリート方式に統一すべき
- `schema.prisma` の `User` モデルが Supabase Auth 移行後も削除されずに残存（`password` フィールド付き）
- `バックログからの場面ジャンプ`未実装（変数スナップショットの設計が必要）
- NodePicker のキーボード操作・詳細プレビュー未実装
- ゲーム公開・共有フロー未実装
- キャラクター検索インデックス（Meilisearch）未実装（現在は Prisma で直接取得）
- E2E・ユニットテスト不足（バックログの push 重複防止ロジックなど未テスト）

### 🟢 優先度低
- `Asset.url` フィールドが `architecture.md` に未記載
- `schema.prisma` の `thumbKeyWebp` / `thumbKeyAvif` フィールドのインデント崩れ
- `prefixText` の computed 内に `console.log` が残存（パフォーマンス影響あり）

## 📋 実装予定・検討中の機能

### 🎬 演出・ビジュアル系

#### キャラクターの登場・退場演出
フェードイン・フェードアウト・スライドインなどをノードプロパティで設定できるようにする。
- 難易度: 中（StageCanvas の描画ロジックに演出タイミングを追加）
- 検討点: 演出の種類をどこまで用意するか（プリセット固定か自由設定か）

#### 揺れエフェクトなどの永続化
現在のビジュアルエフェクトはノード単位でリセットされるが、「ずっと揺れ続ける」などの永続エフェクトを別レイヤーで管理する。
- 難易度: 中
- 検討点: 永続エフェクトを解除するノードの設定方法（「なし」を明示的に設定するか）

#### 任意位置への画像・アニメーション表示
画面の任意位置に日付テロップ・アイテム画像・エフェクト画像などを表示し、アニメーションを付ける機能。
- 難易度: 高（描画レイヤーの追加、座標・サイズ・Zオーダー管理）
- 検討点: キャラクターとして扱う運用で代替可能か要評価。専用機能として作るならオーバーレイレイヤーとして実装

#### 人物のシルエット表示
立ち絵に黒い overlay を被せてシルエット化する。
- 難易度: 低〜中（CSS の `brightness(0)` または Canvas での描画処理）
- 実装案: `CharacterImage` のプロパティとして `silhouette: boolean` を追加

### 🎮 プレイ体験系

#### AUTO 再生機能
テキスト表示完了後に自動で次のノードへ進む。速度調整も可能にする。
- 難易度: 低（タイマーと既存の `advanceWithinNodeOrNext` の組み合わせ）

#### スキップ機能
既読ノードを高速スキップする。未読に差し掛かったら自動停止。
- 難易度: 中（既読フラグの管理が必要。セーブデータとの連携も要検討）

#### バックログからの場面ジャンプ
バックログのエントリをクリックしてその時点に戻る。
- 難易度: 高（変数・フラグのスナップショットを `BacklogEntry` に持たせる必要がある）
- 参照: `architecture.md` の将来対応メモに設計メモあり

### 🖼️ キャラクター・アセット系

#### メッセージウィンドウへの顔アイコン表示
話者キャラの立ち絵から顔部分を切り抜いてウィンドウ横に表示する。
- 難易度: 中〜高（顔領域の定義方法が課題。手動トリミング座標をキャラに持たせるか、自動検出か）
- 検討点: `CharacterImage` に `faceRect: { x, y, w, h }` を追加する設計が現実的

#### キャラクターの顔切り抜き機能
キャラ編集画面で立ち絵画像から顔領域を指定してサムネ化する。上の顔アイコン機能の前提。
- 難易度: 中（トリミング UI + サーバーサイドでの切り抜き処理）

### 🎨 エディタ・制作ツール系

#### 選択肢デザインのカスタマイズ
選択肢ボタンの色・フォント・形状をシナリオ全体設定で変更できるようにする。
- 難易度: 低〜中（`GameUiTheme` に選択肢テーマを追加、既存の仕組みを踏襲）

#### シナリオのテキストエクスポート・インポート
シナリオをプレーンテキスト（CSV・JSON・独自フォーマット）でエクスポートし、一括編集後にインポートできる。
- 難易度: 中（フォーマット設計とバリデーションが要件定義次第）
- ユースケース: 外部翻訳・一括テキスト編集・バックアップ

#### シナリオのフローチャート表示
ノードとその繋がりをビジュアルで確認できる。分岐・ループを俯瞰できる。
- 難易度: 高（グラフ描画ライブラリの導入、ノード座標管理）
- 参考ライブラリ: `@vue-flow/core`、`d3`

#### タイトル画面の作成機能
ゲームにタイトル画面を追加し、背景画像・タイトルロゴ・開始ボタンなどを配置できる。
- 難易度: 中〜高（シナリオとは別の画面定義が必要）

#### タイトル画像からサムネイル生成
タイトル画面の画像をゲームのサムネイルとして自動設定する。検索・一覧に表示。
- 難易度: 低（既存のサムネ生成 Worker を流用）
- 前提: タイトル画面機能の実装後

### 🌐 コミュニティ・プラットフォーム系

#### コメント機能
公開アセット・ゲームに対してコメントを投稿できる。
- 難易度: 中（`Comment` モデル追加・API・フロントUI）
- 検討点: 通知機能との連携、モデレーション

#### ランキング機能
お気に入り数・コメント数・プレイ時間などを集計してランキング表示する。
- 難易度: 中〜高（プレイ時間の計測・集計バッチが必要）
- 検討点: プレイ時間は `GameSession` のようなモデルで記録する設計が必要

## 💡 Claude からの追加アイデア

#### BGM・SE のフェードイン／アウト
ノード切り替え時に音楽がブツ切りにならず自然につながる。
- 難易度: 低（Web Audio API の `gainNode` で制御）

#### ノードのコメント・メモ機能
エディタ上でノードにメモを残せる。シナリオ管理・引き継ぎに便利。
- 難易度: 低（`GameNode` に `memo: String?` を追加するだけ）

#### 多言語対応（i18n）
テキストを言語別に持てるようにして、プレイ時に言語切替ができる。
- 難易度: 高（データモデルの大幅変更が必要）
- 検討点: 需要と優先度の整理が先

#### プレイ統計ダッシュボード（制作者向け）
ゲーム制作者が自分のゲームのプレイ回数・平均プレイ時間・どのノードで離脱しているかを確認できる。
- 難易度: 高（イベントトラッキングの仕組みが必要）
- ランキング機能と組み合わせると相乗効果あり

#### シナリオのバージョン管理
ゲームの過去バージョンを保存・復元できる。誤操作によるデータ消失を防ぐ。
- 難易度: 高（スナップショット設計が複雑）
- 現実的な代替案: 「公開前に確定版として保存」程度の簡易版から始める

#### ゲームのコレクション・プレイリスト機能
複数のゲームをまとめてシリーズとして管理・公開できる。
- 難易度: 中（`GameCollection` モデルの追加）
